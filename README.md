# ⚡ WATT Protocol
**Energy to Earn** | Born in Africa, Built for the World

WATT Protocol is an open-source blockchain infrastructure that bridges **real-world renewable energy generation** with **verifiable ESG utility**. By deploying cryptographically signed smart meters on solar, wind, and microgrid installations, we tokenise verified kilowatt-hours into `$WATT`—turning clean energy production into transparent, on-chain impact.

---

## 🔋 How the Utility Works
| Step | Action | On-Chain Result |
|:---:|:---|:---|
| 🏠 **Install** | Smart meters deployed on renewable energy systems (households & commercial) | Device registered to owner wallet |
| 📡 **Verify** | Telemetry signed at the edge & transmitted to protocol backend | Data immutably logged & validated |
| 🪙 **Mint** | `$WATT` issued proportionally to verified kWh generation | ERC-20 tokens minted to producer |
| 🌍 **Impact** | Tokens power ESG tracking, community funding, & on-chain governance | Transparent climate impact ledger |

> 💡 *$WATT is not a speculative asset. It is a utility token backed by audited renewable generation, designed for ESG compliance, carbon credit verification, and community impact funding.*

## 📦 Protocol Architecture
WATT Protocol is built as a modular, auditable stack. Each layer uses a purpose-built license to maximise transparency while protecting strategic assets.

| Repository | License | Purpose |
|:---|:---|:---|
| [`📜 contracts`](https://github.com/watt-protocol/contracts) | MIT | Core smart contracts (Base chain): $WATT, treasury, impact fund, governance |
| [`🔌 firmware`](https://github.com/watt-protocol/firmware) | Apache 2.0 | ESP32/Raspberry Pi meter firmware, cryptographic signing & edge telemetry |
| [`🖥️ server`](https://github.com/watt-protocol/server) | AGPL v3 | Backend verification, minting orchestration & household dashboard |
| [`📊 network-dashboard`](https://github.com/watt-protocol/network-dashboard) | AGPL v3 | Live public stats: total kWh, active meters, geographic distribution, ESG impact |
| [`📚 sdk`](https://github.com/watt-protocol/sdk) | MIT | JS/TS SDK for developers, installers, & ESG verifiers |
| [`📖 docs`](https://github.com/watt-protocol/docs) | MIT/CC BY | Developer guides, API references, meter integration specs |
| [`🔍 audits`](https://github.com/watt-protocol/audits) | CC BY | Public security audit reports & vulnerability disclosures |

## 🌍 Open Source & Licensing Strategy
We operate under a **layered open-source matrix** to satisfy grant funders, ensure reproducibility, and protect long-term sustainability:
- ✅ **Core Protocol & SDK** → Permissive licenses for maximum adoption & third-party integration
- ✅ **Backend & Dashboards** → AGPL v3 to prevent uncompensated commercial forking of verification logic
- ✅ **Future Data & Hardware** → ODbL & CERN-OHL-S for climate research, grid planning, and open manufacturing
- 🔒 **Brand, Website & KYC Data** → Proprietary to protect community identity, operational security, and user privacy

*Full licensing matrix & funder compliance docs available in [`watt-protocol/.github`](https://github.com/watt-protocol/.github).*
